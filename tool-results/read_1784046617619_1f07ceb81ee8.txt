   488тЖТ## 4. P0: Core Corrections (Accuracy)
   489тЖТ
   490тЖТ### 4.1 Atmospheric Correction for EDM
   491тЖТ
   492тЖТ**File**: `src/lib/survey/corrections/atmospheric.ts`
   493тЖТ
   494тЖТ**Formula** (IAO/ISO standard):
   495тЖТ```
   496тЖТCorrected distance = Raw distance ├Ч N0 / N
   497тЖТ
   498тЖТWhere:
   499тЖТ  N = (n_ref - 1) ├Ч 10^6  (reference refractivity)
   500тЖТ  N0 = (n_obs - 1) ├Ч 10^6 (observed refractivity)
   501тЖТ
   502тЖТFor standard conditions (t=20┬░C, P=1013.25 hPa, e=0):
   503тЖТ  N0 = 273.82 (typical for most EDM instruments)
   504тЖТ
   505тЖТFor observed conditions:
   506тЖТ  N_obs = (n - 1) ├Ч 10^6
   507тЖТ  n = 1 + (n_group - 1) ├Ч ╬╗_ref / ╬╗_obs  (for phase measurement)
   508тЖТ  
   509тЖТSimplified (most instruments):
   510тЖТ  N = (287.604 + 1.6288 ├Ч ╬╗^-2 + 0.0136 ├Ч ╬╗^-4) ├Ч P / (273.15 + t)
   511тЖТ      - (11.27 ├Ч e) / (273.15 + t)
   512тЖТ      
   513тЖТ  Where ╬╗ = carrier wavelength in ╬╝m (0.6328 for HeNe, 0.850 for IR)
   514тЖТ  
   515тЖТ  e = humidity ├Ч 6.112 ├Ч exp(17.62 ├Ч t / (243.12 + t)) / 100  (Magnus formula)
   516тЖТ```
   517тЖТ
   518тЖТ**Implementation requirements**:
   519тЖТ- Support both HeNe (0.6328 ╬╝m) and IR (0.850 ╬╝m) EDM wavelengths
   520тЖТ- Accept temperature (┬░C), pressure (hPa), humidity (%)
   521тЖТ- Auto-compute partial water vapor pressure via Magnus formula
   522тЖТ- Return both correction factor and corrected distance
   523тЖТ- Include ppm deviation from reference conditions
   524тЖТ- Store raw + corrected for audit trail
   525тЖТ
   526тЖТ**Validation**:
   527тЖТ- At standard conditions (20┬░C, 1013.25 hPa, 0% humidity): correction = 0 ppm
   528тЖТ- At Nairobi (25┬░C, 840 hPa, 60% humidity): correction тЙИ -22 ppm (distance reduced)
   529тЖТ- At Mombasa (32┬░C, 1010 hPa, 80% humidity): correction тЙИ -10 ppm
   530тЖТ
   531тЖТ### 4.2 Curvature & Refraction Correction
   532тЖТ
   533тЖТ**File**: `src/lib/survey/corrections/curvature-refraction.ts`
   534тЖТ
   535тЖТ**Formula**:
   536тЖТ```
   537тЖТC&R correction = -0.0675 ├Ч D┬▓  (meters, D in km)
   538тЖТ
   539тЖТThis is applied to height difference:
   540тЖТ  ╬ФH_corrected = SD ├Ч sin(VA) + (1 - k) / (2R) ├Ч D┬▓
   541тЖТ
   542тЖТWhere:
   543тЖТ  k = coefficient of refraction (0.13 for Kenya, 0.14 temperate)
   544тЖТ  R = mean radius of Earth at latitude = 6370 km (equatorial) to 6357 km (polar)
   545тЖТ  R at Kenya latitude (0┬░-1┬░S) тЙИ 6378 km (close to equatorial)
   546тЖТ  D = horizontal distance in km
   547тЖТ  
   548тЖТCombined formula:
   549тЖТ  ╬ФH = SD ├Ч sin(VA) + (ih - th) + C&R
   550тЖТ  Where C&R = (1 - k) / (2R) ├Ч D┬▓
   551тЖТ  = (1 - 0.13) / (2 ├Ч 6378000) ├Ч D┬▓
   552тЖТ  = 0.0675 ├Ч D┬▓  (D in km, result in meters)
   553тЖТ```
   554тЖТ
   555тЖТ**Implementation requirements**:
   556тЖТ- Accept slope distance, vertical angle, instrument height, target height
   557тЖТ- Use latitude-dependent Earth radius (WGS84 ellipsoid)
   558тЖТ- Configurable refraction coefficient (default 0.13 for Kenya)
   559тЖТ- Return corrected height difference with C&R component separately
   560тЖТ- Flag observations where C&R > 10mm (recommendation: apply for all lines >200m)
   561тЖТ
   562тЖТ### 4.3 Grid Scale Factor
   563тЖТ
   564тЖТ**File**: `src/lib/survey/corrections/grid-scale-factor.ts`
   565тЖТ
   566тЖТ**Formula** (UTM):
   567тЖТ```
   568тЖТPoint scale factor at latitude ╧Ж, longitude ╬╗:
   569тЖТ  k = k0 ├Ч [1 + (╬╗ - ╬╗0)┬▓ / 2 ├Ч cos┬▓(╧Ж) ├Ч (1 + e'┬▓ ├Ч cos┬▓(╧Ж) + ...)]
   570тЖТ
   571тЖТSimplified (sufficient for 2nd order):
   572тЖТ  k = k0 ├Ч (1 + E'┬▓ / (2 ├Ч Rm┬▓))
   573тЖТ
   574тЖТWhere:
   575тЖТ  k0 = 0.9996 (UTM central scale factor)
   576тЖТ  E' = easting from central meridian = E - 500000
   577тЖТ  Rm = mean radius of curvature at latitude ╧Ж
   578тЖТ  Rm = тИЪ(╧Б ├Ч ╬╜)
   579тЖТ  ╧Б = a(1 - e┬▓) / (1 - e┬▓ ├Ч sin┬▓(╧Ж))^(3/2)  (meridional radius)
   580тЖТ  ╬╜ = a / тИЪ(1 - e┬▓ ├Ч sin┬▓(╧Ж))                (prime vertical radius)
   581тЖТ  
   582тЖТFor WGS84/Arc 1960:
   583тЖТ  a = 6378137.0 m (semi-major axis)
   584тЖТ  f = 1/298.257223563
   585тЖТ  e┬▓ = 2f - f┬▓ = 0.00669437999014
   586тЖТ
   587тЖТFor Cassini-Soldner (Kenya cadastral):
   588тЖТ  k = 1 + E'┬▓ / (2 ├Ч Rm┬▓)  (no k0 central factor)
   589тЖТ```
   590тЖТ
   591тЖТ**Implementation requirements**:
   592тЖТ- Support both UTM (Zone 36S, 37S) and Cassini-Soldner projections
   593тЖТ- Compute point scale factor from UTM easting/northing
   594тЖТ- Compute line scale factor (average of endpoints or Simpson's rule)
   595тЖТ- Include arc-to-chord correction for grid bearings
   596тЖТ- Return scale factor, corrected distance, and convergence angle
   597тЖТ
   598тЖТ### 4.4 Sea Level / Ellipsoid Reduction
   599тЖТ
   600тЖТ**File**: `src/lib/survey/corrections/sea-level-reduction.ts`
   601тЖТ
   602тЖТ**Formula**:
   603тЖТ```
   604тЖТReduced distance = Measured distance ├Ч R / (R + h)
   605тЖТ
   606тЖТWhere:
   607тЖТ  R = mean radius of Earth at latitude (use same Rm as grid scale factor)
   608тЖТ  h = mean height of the measured line above the ellipsoid
   609тЖТ
   610тЖТFor Kenya:
   611тЖТ  h_ellipsoid тЙИ h_orthometric + N (geoid height)
   612тЖТ  N (geoid undulation) for Kenya тЙИ -10 to -20m (varies by location)
   613тЖТ  h_orthometric from leveling тЙИ 0-5000m (most work at 1000-2500m)
   614тЖТ```
   615тЖТ
   616тЖТ**Implementation requirements**:
   617тЖТ- Accept measured distance and mean height of line
   618тЖТ- Optional geoid undulation input (default Kenya EGM96 values)
   619тЖТ- Apply reduction before grid scale factor
   620тЖТ- Full reduction chain: slope тЖТ horizontal тЖТ sea level тЖТ grid
   621тЖТ- Return each stage separately for audit trail
   622тЖТ
   623тЖТ### 4.5 Unified Correction Pipeline
   624тЖТ
   625тЖТ**File**: `src/lib/survey/pipeline/correction-pipeline.ts`
   626тЖТ
   627тЖТ**Processing order** (this order matters!):
   628тЖТ```
   629тЖТ1. EDM Instrument Constant     тЖТ Add constant to raw distance
   630тЖТ2. Atmospheric Correction       тЖТ Correct raw slope distance for temp/pressure/humidity
   631тЖТ3. Slope Reduction              тЖТ Convert slope distance to horizontal distance
   632тЖТ4. Curvature & Refraction       тЖТ Correct height difference for C&R
   633тЖТ5. Sea Level Reduction          тЖТ Reduce horizontal distance to ellipsoid
   634тЖТ6. Grid Scale Factor            тЖТ Scale ellipsoid distance to grid distance
   635тЖТ7. Grid Convergence             тЖТ Correct bearings for projection convergence
   636тЖТ```
   637тЖТ
   638тЖТEach stage must:
   639тЖТ- Input: observation + previous stage output
   640тЖТ- Output: corrected value + correction component + uncertainty contribution
   641тЖТ- Log: what was applied, input values, output values
   642тЖТ- Flag: if any correction exceeds configurable thresholds
   643тЖТ

   646тЖТ## 5. P1: Traverse Engine & Error Propagation
   647тЖТ
   648тЖТ### 5.1 Traverse Engine
   649тЖТ
   650тЖТ**File**: `src/lib/survey/traverse/engine.ts`
   651тЖТ
   652тЖТ**Implementation**:
   653тЖТ1. Accept raw observations (angles + distances) + control points
   654тЖТ2. Run through correction pipeline (all P0 corrections)
   655тЖТ3. Compute preliminary coordinates
   656тЖТ4. Compute misclosure (angular + linear)
   657тЖТ5. Evaluate misclosure ratio against required order
   658тЖТ6. Apply adjustment (Bowditch for 3rd/4th order, Least Squares for 1st/2nd)
   659тЖТ7. Compute adjusted coordinates with uncertainties
   660тЖТ8. Generate computation sheet data
   661тЖТ
   662тЖТ### 5.2 Error Propagation Engine
   663тЖТ
   664тЖТ**File**: `src/lib/survey/error-propagation/engine.ts`
   665тЖТ
   666тЖТ**Methods**:
   667тЖТ1. **Variance propagation**: For any function f(xтВБ, xтВВ, ..., xтВЩ):
   668тЖТ   ╧Г┬▓_f = ╬г(тИВf/тИВxс╡в)┬▓ ├Ч ╧Г┬▓_xс╡в + 2 ├Ч ╬г(тИВf/тИВxс╡в)(тИВf/тИВxт▒╝) ├Ч ╧Г_xс╡вxт▒╝
   669тЖТ
   670тЖТ2. **Covariance matrix**: Full variance-covariance matrix from least squares
   671тЖТ
   672тЖТ3. **Error ellipse**: From covariance matrix sub-matrix:
   673тЖТ   - Semi-major: a = ╧ГтВАтИЪ(0.5 ├Ч (╧Г┬▓_E + ╧Г┬▓_N + тИЪ((╧Г┬▓_E - ╧Г┬▓_N)┬▓ + 4╧Г┬▓_EN)))
   674тЖТ   - Semi-minor: b = ╧ГтВАтИЪ(0.5 ├Ч (╧Г┬▓_E + ╧Г┬▓_N - тИЪ((╧Г┬▓_E - ╧Г┬▓_N)┬▓ + 4╧Г┬▓_EN)))
   675тЖТ   - Orientation: ╬╕ = 0.5 ├Ч atan2(2╧Г_EN, ╧Г┬▓_E - ╧Г┬▓_N)
   676тЖТ
   677тЖТ4. **Reliability**: Redundancy numbers, internal/external reliability
   678тЖТ
   679тЖТ### 5.3 Least Squares Adjustment
   680тЖТ
   681тЖТ**File**: `src/lib/survey/traverse/least-squares.ts`
   682тЖТ
   683тЖТ**Method**: Variation of coordinates (parametric adjustment)
   684тЖТ- Design matrix A from observation equations
   685тЖТ- Weight matrix P from a priori standard deviations
   686тЖТ- Normal equations: N = Aс╡АPA, solution: ╬┤x = NтБ╗┬╣Aс╡АPl
   687тЖТ- A posteriori variance factor: ╧Г╠ВтВА┬▓ = vс╡АPv / (n - u)
   688тЖТ- Covariance matrix: C_xx = ╧Г╠ВтВА┬▓ ├Ч NтБ╗┬╣
   689тЖТ- Statistical tests: Chi-square test on ╧Г╠ВтВА┬▓, local test on residuals
   690тЖТ

   736тЖТ## 7. Database Load Reduction Strategy
   737тЖТ
   738тЖТ### 7.1 Offline-First Architecture
   739тЖТ
   740тЖТ```
   741тЖТCLIENT (Field)                    SERVER (Office)
   742тЖТтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФА                     тФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФА
   743тЖТIndexedDB тЖРтФАтФАтФАтФА Sync API тФАтФАтФАтФАтЖТ PostgreSQL
   744тЖТ(All field work)   (Batch)       (Computation & docs)
   745тЖТ```
   746тЖТ
   747тЖТ- **All field data collection happens offline** тАФ IndexedDB stores observations
   748тЖТ- **Sync only when connected** тАФ batch upload at end of day
   749тЖТ- **Server does heavy computation** тАФ traverse adjustment, document generation
   750тЖТ- **Results cached** тАФ adjusted coordinates stored both client and server
   751тЖТ
   752тЖТ### 7.2 Caching Strategy
   753тЖТ
   754тЖТ| Data Type | Cache Location | TTL | Invalidation |
   755тЖТ|-----------|---------------|-----|-------------|
   756тЖТ| Coordinate system params | In-memory LRU | 24h | On parameter update |
   757тЖТ| Datum transformation params | In-memory LRU | 24h | On update |
   758тЖТ| Project metadata | In-memory LRU | 5min | On project update |
   759тЖТ| Survey computation results | Redis | 1h | On re-computation |
   760тЖТ| Generated documents | File system | Permanent | On regeneration |
   761тЖТ| User session data | Redis | 30min | On logout |
   762тЖТ| Reference data (beacons, control) | In-memory LRU | 1h | On update |
   763тЖТ
   764тЖТ### 7.3 Query Optimization
   765тЖТ
   766тЖТ1. **Connection Pooling**: PgBouncer in transaction mode
   767тЖТ   - Max server connections: 50
   768тЖТ   - Max client connections: 500
   769тЖТ   - Reserve pool: 5 (for admin)
   770тЖТ
   771тЖТ2. **Prepared Statements**: For all frequently-used queries
   772тЖТ
   773тЖТ3. **Batch Operations**:
   774тЖТ   - Field sync: Single INSERT with array of observations
   775тЖТ   - Coordinate updates: UNNEST-based bulk update
   776тЖТ   - Document generation: Queue with BullMQ + Redis
   777тЖТ
   778тЖТ4. **Read-Heavy Query Offloading**:
   779тЖТ   - Project listing тЖТ Materialized view, refresh on change
   780тЖТ   - Coordinate queries тЖТ PostGIS spatial index
   781тЖТ   - Document listing тЖТ Separate read-only connection
   782тЖТ
   783тЖТ5. **Write Optimization**:
   784тЖТ   - Observation sync тЖТ COPY protocol for bulk inserts
   785тЖТ   - Audit log тЖТ Async insert via queue (don't block main query)
   786тЖТ   - Status updates тЖТ Optimistic concurrency with version field
   787тЖТ
   788тЖТ### 7.4 Heavy Computation Offloading
   789тЖТ
   790тЖТ```
   791тЖТтФМтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФР     тФМтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФР     тФМтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФР
   792тЖТтФВ  Next.js тФВтФАтФАтФАтФАтЖТтФВ  BullMQ  тФВтФАтФАтФАтФАтЖТтФВ  Worker      тФВ
   793тЖТтФВ  API     тФВ     тФВ  Queue   тФВ     тФВ  (Python/TS) тФВ
   794тЖТтФВ  (fast)  тФВ     тФВ  + Redis тФВ     тФВ  (compute)   тФВ
   795тЖТтФФтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФШ     тФФтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФШ     тФФтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФШ
   796тЖТ     тЖС                                   тФВ
   797тЖТ     тФФтФАтФАтФАтФАтФАтФАтФАтФАтФА Result via DB тФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФШ
   798тЖТ```
   799тЖТ
   800тЖТ- **Traverse adjustment** тЖТ Queue, compute async, poll for result
   801тЖТ- **Document generation** тЖТ Queue, compute async, download when ready
   802тЖТ- **Area computation** тЖТ Fast enough for inline (<100ms)
   803тЖТ- **COGO** тЖТ Fast enough for inline (<50ms)
   804тЖТ
   805тЖТ### 7.5 Database Schema Optimization
   806тЖТ
   807тЖТ- **Partitioning**: `Observation` table by month (for large projects)
   808тЖТ- **Partial indexes**: `WHERE status = 'ACTIVE'` for project listing
   809тЖТ- **JSONB GIN indexes**: On `correctionsLog`, `correctionParams`
   810тЖТ- **Covering indexes**: Include frequently-accessed columns in index
